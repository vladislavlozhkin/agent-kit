# Agent Kit

Набор инструментов и оберток (wrappers) для интеграции AI-моделей (в частности Gemini) в рабочий процесс командной строки. Позволяет превратить обычный CLI в агента с памятью и контекстом проекта.

## Возможности

- **Сохранение контекста:** Автоматическое ведение логов диалогов в Markdown.
- **Интеграция с проектом:** Скрипты устанавливаются локально в директорию `.claude` вашего проекта.
- **Гибкость:** Поддержка различных моделей через переменные окружения.

## Компоненты

### Gemini Agent (`scripts/gemini`)
Скрипты для ведения непрерывного диалога с моделью Gemini.

| Скрипт | Описание |
| :--- | :--- |
| `new.sh` | Начать новую сессию (диалог). Создает новый лог-файл. |
| `continue.sh` | Продолжить текущую сессию. Добавляет ваш запрос и ответ модели в последний лог. |

**Требования:**
- Установленная утилита `gemini` в PATH (или алиас).
- `git`, `curl` или `wget`.

## Установка

Чтобы добавить инструменты агента в ваш текущий проект, выполните следующую команду в корне проекта:

```bash
curl -fsSL https://raw.githubusercontent.com/vladislavlozhkin/agent-kit/main/install.sh | bash
```

Скрипт выполнит следующие действия:
1. Создаст директорию `.claude` (если она отсутствует).
2. Загрузит скрипты агента в `.claude/scripts/gemini/`.
3. Загрузит системные промпты в `.claude/commands/`.
4. Подготовит директорию для логов `.claude/logs/gemini/`.

## Использование

После установки скрипты будут доступны в папке `.claude`.

### Начало нового диалога
Используйте `new.sh` для старта новой темы. Это создаст новый файл лога.

```bash
./.claude/scripts/gemini/new.sh "Привет! Проанализируй структуру этого проекта."
```

### Продолжение диалога
Используйте `continue.sh`, чтобы продолжить общение в рамках последней сессии. Скрипт отправит всю историю текущего диалога модели.

```bash
./.claude/scripts/gemini/continue.sh "А теперь предложи рефакторинг для main.ts."
```

### Логирование
Все диалоги сохраняются в `.claude/logs/gemini/`.
- Файлы именуются по дате: `YYYY-MM-DD_HH-MM-SS.md`.
- Создается симлинк `latest.md`, указывающий на последнюю активную сессию.

## Конфигурация

Вы можете настроить используемую модель через переменную окружения `GEMINI_MODEL`.

| Переменная | По умолчанию | Описание |
| :--- | :--- | :--- |
| `GEMINI_MODEL` | `pro` | Идентификатор модели (например, `flash`, `pro`, `gemini-1.5-pro`) |

**Примеры:**

```bash
# Использовать модель flash для одной команды
GEMINI_MODEL="flash" ./.claude/scripts/gemini/new.sh "Быстрый вопрос"

# Установить для текущей сессии терминала
export GEMINI_MODEL="gemini-1.5-pro-latest"
./.claude/scripts/gemini/continue.sh "Сложная задача"
```
