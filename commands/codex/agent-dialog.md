## Цель

Запустить саб-агента `codex-dialog` для автономного P2P диалога с Codex. 

## Как вызвать

Используй Task tool с `subagent_type: "codex-dialog"`:

```
Task tool:
  subagent_type: "codex-dialog"
  prompt: "Оцени стратегии кэширования: Redis vs in-memory для API с 10k req/min"
```

Саб-агент:
1. Начнёт сессию диалога с Codex
2. Проведёт несколько раундов обсуждения
3. Отследит бюджет прогресса и попытки синтеза
4. Вернёт результат: КОНСЕНСУС или ЭСКАЛАЦИЯ

## Понимание результатов

### КОНСЕНСУС

Достигнуто согласие. Результат включает:
- Резюме обсуждения
- Что Claude принял от Codex
- Согласованную позицию

**Действие:** Используй согласованную позицию в ответе пользователю.

### ЭСКАЛАЦИЯ

Консенсус не достигнут. Результат включает:
- Обе позиции с аргументацией
- Конкретную точку разногласия
- Рекомендацию (если есть)

**Действие:** Представь обе позиции пользователю для его решения.

## Пример

```
User: "Спроектируй стратегию кэширования для нашего API"

Claude: "Проконсультируюсь с Codex по подходу к кэшированию."

[Вызов codex-dialog агента с: "Оцени стратегии кэширования для API с 10k req/min, смешанная нагрузка чтение/запись"]

[Агент возвращает консенсус по гибридному подходу Redis + CDN]

Claude: "После консультации с Codex, рекомендуем гибридный подход..."
```
