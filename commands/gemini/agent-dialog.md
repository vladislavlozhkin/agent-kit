## Цель

Запустить саб-агента `gemini-dialog` для автономного P2P диалога с Gemini.
## Как вызвать

Используй Task tool с `subagent_type: "gemini-dialog"`:

```
Task tool:
  subagent_type: "gemini-dialog"
  prompt: "Оцени варианты базы данных: PostgreSQL vs MongoDB для нашего случая"
```

Саб-агент:
1. Начнёт сессию диалога с Gemini
2. Проведёт несколько раундов обсуждения
3. Отследит бюджет прогресса и попытки синтеза
4. Вернёт результат: КОНСЕНСУС или ЭСКАЛАЦИЯ

## Понимание результатов

### КОНСЕНСУС

Достигнуто согласие. Результат включает:
- Резюме обсуждения
- Что Claude принял от Gemini
- Согласованную позицию

**Действие:** Используй согласованную позицию в ответе пользователю.

### ЭСКАЛАЦИЯ

Консенсус не достигнут. Результат включает:
- Обе позиции с аргументацией
- Конкретную точку разногласия
- Рекомендацию (если есть)

**Действие:** Представь обе позиции пользователю для его решения.

## Пример

```
User: "Помоги выбрать между микросервисами и монолитом"

Claude: "Проконсультируюсь с Gemini по архитектурному подходу."

[Вызов gemini-dialog агента с: "Микросервисы vs монолит для e-commerce платформы, команда 5 разработчиков"]

[Агент возвращает консенсус по модульному монолиту]

Claude: "После консультации с Gemini, рекомендуем подход модульного монолита..."
```
